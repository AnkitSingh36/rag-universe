Integration
🔌 Step 2 — Register & Wire Up

Two more files to touch after creating your class.

2a
Register in rag_core/__init__.py
Add your import and register in the ALL_RAG_TYPES dictionary. This is all the app needs to know your RAG exists.
from .my_custom_rag import MyCustomRAG ALL_RAG_TYPES["🆕 My Custom RAG"] = MyCustomRAG
2b
Add a pipeline diagram in rag_diagrams/pipeline.py
Each RAG type has a visual HTML pipeline shown on the Encyclopedia page. Use the helper functions to build yours in under 5 lines.
def diagram_my_custom_rag(): content = _row( _box("Query", "❓", "#FF6B6B"), _arrow(), _box("My Logic", "🔧", "#6366F1"), _arrow(), _box("Answer", "💬", "#10B981"), ) return _wrap(content, "🆕 My Custom RAG", "Brief description") DIAGRAM_MAP["My Custom RAG"] = diagram_my_custom_rag
2c
Run the app and test
Your RAG now appears everywhere automatically. Open Page 3 (Test Any RAG), pick your type from the dropdown, and run a query.
streamlit run app.py # Navigate to ⭐ Test Any RAG → select "🆕 My Custom RAG"
Inspiration
💡 5 RAG Ideas to Build Next

All five are achievable in under 200 lines of Python. Difficulty estimated relative to Naive RAG.

📅
Temporal RAG
Filter retrieved chunks by date before ranking. Useful for news bots where recent articles should outrank old ones, even if semantically similar.
Difficulty: ⭐ Easy — add a date field to Document, sort before top-k
🌐
Multilingual RAG
Translate the query to English before embedding, retrieve English chunks, then translate the answer back. Handles corpora in multiple languages.
Difficulty: ⭐⭐ Medium — wrap with translate() calls using deep-translator
🔐
Permissioned RAG
Tag each chunk with an ACL (access control list). At retrieval time, filter to only chunks the current user is allowed to see.
Difficulty: ⭐⭐ Medium — add metadata field, filter in retrieve()
🖼️
Multimodal RAG
Use CLIP to embed both images and text queries into the same vector space. Retrieve images or diagrams alongside text chunks.
Difficulty: ⭐⭐⭐ Hard — requires CLIP model + image pipeline
🧩
Ensemble RAG
Run Naive + Hybrid + Rerank in parallel, then vote on which chunks appear most across all three. The intersection = high-confidence retrieval.
Difficulty: ⭐⭐ Medium — instantiate 3 RAGs, merge chunk sets
Quality Assurance
🧪 Testing Checklist

Run through these checks before calling your implementation complete.

Correctness
add_documents() indexes all chunks without errors
retrieve() returns Document objects with .score set
ask() returns a RAGResult with non-empty .answer
Scores are in range [0, 1] (normalize if needed)
retrieve() respects the top_k parameter
Works with all 5 sample topics
Edge Cases
Empty query string doesn't crash
Query with no matching chunks returns graceful answer
top_k larger than total chunks doesn't crash
Very long document (10,000+ chars) chunks correctly
Works with Ollama enabled AND disabled
Second call to add_documents() adds (not replaces) docs
Real Examples
📚 How the Built-In RAGs Are Structured

Study these patterns from the actual implementations in rag_core/.

🔵 naive_rag.py — Simplest possible

  • add_documents: chunk → embed → append to list
  • retrieve: embed query → L2 distance against all embeddings → top-k
  • ask: retrieve → demo_generate
  • ~80 lines total

🟣 hybrid_rag.py — Two indexes

  • add_documents: build both FAISS index AND BM25 index
  • retrieve: run both → label results → RRF merge
  • Documents carry a .retrieval_method label
  • ~140 lines total

🟢 conversational_rag.py — Stateful

  • Adds self.history = [] to maintain turns
  • retrieve: prepend last N turns to query string
  • ask: append Q+A to history after each call
  • ~120 lines total

🔶 rerank_rag.py — Two-stage

  • retrieve stage 1: top-20 via vector search
  • retrieve stage 2: cross-encoder scores all 20
  • Returns top-3 by cross-encoder score
  • ~110 lines total
